Embian APM, e2eMon의 특징

운영중인 시스템이 갑자기 느려지거나 에러가 발생하셨습니까? e2eMon은 서비스 도메인 수준에서 빨리 문제점을 진단하고 추적 할 수 있도록, 여러가지 정보를 다양한 표현 방식을 이용해 출력하고 있습니다.

  • 애플리케이션 모니터링
  • 에러와 장애 징후 탐지
  • 주요 기능

애플리케이션 모니터링

애플리케이션의 성능(페이지 로딩 속도, 에러율, 느린 트랜젝션, 애플리케이션과의 관계 등)을 Dashboard를 통해 한눈에 모니터링 할 수 있습니다.

트랜젝션 응답 시간

트랜젝션 응답 시간은 클라이언트에서 보낸 Request를 서버가 받은 순간부터, 최종 Response를 전송하는 순간까지의 총 소요 시간을 나타냅니다. 처리 과정(Transaction)은 애플리케이션간 또는 서버간 통신, 외부 API 호출, 케싱, 애플리케이션 코드 수행 시간, 데이터베이스 쿼리, 비동기 통신의 실제 코드 수행시간 등, 시간이 소요되는 모든 과정을 포함해서 산출됩니다.

그래프로 표현되는 애플리케이션의 성능

다양한 형태의 그래프를 이용해 애플리케이션의 성능과 관련된 통계를 시각적으로 표현합니다. 다양한 그래프들은 단순히 현재 시스템의 성능을 확인하는 용도로 사용할 수도 있고, 전체적으로 시스템의 취약점, 문제점을 찾고 해결하기 위한 시작점으로써 활용될 수 있습니다..

트랜젝션 중 가장 오래 걸린 곳

트랜젝션을 처리하는데 걸린 총 시간은 얼마이고 처리 과정에서 가장 시간이 오래걸린 부분이 어디인지, 처리 결과를 세부적으로 제공합니다. 각각의 과정은 다시 코드 수준까지 들어가서 정보를 확인 할 수 있으며 이를 바탕으로 어떤 부분에서 시간이 소요되었는지를 그 포인트를 특정할 수 있습니다.

외부 서비스의 성능

e2eMon은 내부 시스템 뿐만 아니라 외부 시스템으로의 원격 호출에 대한 성능 측정 정보도 제공합니다. 외부 시스템으로 Request를 보낸 순간부터 Response를 받은 순간까지의 시간을 측정하여 전체 트랜젝션에 미치는 영향을 알 수 있게 도와줍니다.

트랜젝션 추적

트랜젝션의 처리과정을 처음부터 끝까지 추적하여 그 정보를 다이어그램으로 시각화해서 보여줍니다. 트랜젝션의 구체적인 처리과정을 코드를 들여다 보지 않고도 파악할 수 있으며, 에러가 발생한 부분이나 Slow Query, 잠재적인 위험 요소등을 쉽게 찾을 수 있습니다.

애플리케이션간 흐름 추적

트랜젝션은 하나의 애플리케이션에서 모두 처리될 수도 있지만, 여러개의 애플리케이션 또는 서버를 거치면서 처리되기도 합니다. e2eMon은 서로 다른 애플리케이션이나 서버사이의 트랜젝션 흐름도 추적할 수 있어 분산 시스템 기반의 서비스에서 그 진가를 발휘합니다.

주요 비지니스 트랜젝션 관리 [개발예정]

비지니스 상 중요한 트랜젝션들은 따로 등록해 두고, 지속적으로 관리하고 세부 내용을 추적 할 수 있습니다. 또한 트랜젝션들을 그룹핑해서 여러개의 트랜젝션들로 구성된 논리적인 액션을 생성할 수 있습니다.

트랜젝션 상세보기

트랜젝션 상세보기 페이지를 통해 하나의 트랜젝션에 대한 자세하고 다양한 정보(파라미터, Call별 응답시간, 호출 지연 시간 등)를 열람할 수 있습니다. 또한 그래프를 통해 트랜젝션의 전체적인 흐름을 시각적으로 보여줌으로써 트랜젝션에 대한 이해도를 높여줍니다.

스택 추적기 [개발예정]

e2eMon의 스택 추적기를 통해 애플리케이션의 처리 과정을 코드 수준으로 자세하게 들여다 볼 수 있습니다. 스택 추적기는 시슷템 성능에 거의 영향을 미치지 않도록 설계되었습니다.

Service Map

여러대의 서버들로 구성된 분산 시스템 환경에서 시스템의 구조를 맵형태로 보여주는 Server Map을 통해, 시스템 구조에 대한 이해와 시스템의 성능 및 취약점등을 쉽게 파악할 수 있습니다.

에러와 장애 징후 탐지 [개발예정]

신속하게 시스템의 문제를 인지하고 대처할 수 있도록 에러 경고와 위험 임계치를 설정 할 수 있습니다. 경고 데이터는 APDex, 에러율, 응답시간, 주요 트랜젝션등 다양한 값을 대상으로 설정이 가능합니다.

경고 정책

경고 정책은 애플리케이션 그룹별로 위험 수준 임계치를 따로 설정할 수 있습니다. 위험 수준은 "1단계","2단계","장애 발생" 단계로 나눌 수 있으며, 경고는 가청, 가시, SMS, E-mail등 다양한 경로를 통해 이루어 집니다.

주요 기능

Dashboard 개인화 기능

Dashboard를 통해 e2eMon를 사용하는 시스템 운영자는 전체적인 서비스의 구조, 품질, 에러 상태등을 한눈에 파악할 수 있습니다. 그리고 운영자별로 자신이 보고 싶은 데이터를 볼 수 있도록 Dashboard의 컨텐츠 및 화면 배치를 편집하고 관리할 수 있습니다.

윈도우 타임 선택 기능

e2eMon은 최근 5분동안의 데이터를 기반으로 시스템의 성능을 측정합니다. 이 구간을 윈도우 타임이라고 하는데, 서비스의 특징이나 운영자의 운영 방침에 따라 윈도우 타임을 수정해야 할 경우 대비해 윈도우 타임 선택 기능을 제공합니다.

실시간 모니터링 기능

모니터링을 위한 모든 데이터는 실시간으로 수집 및 가공되고 있으며, 웹클라이언트는 성능 측정값의 변화를 자동 업데이트 기능을 통해 확인 할 수 있습니다.

트랜젝션 갯수

윈도우 타임내에 발생한 트랜젝션의 갯수를 "정상","경고 1","경고 2","에러" 단계 별로 각각 분류해서 출력합니다. 운영자는 실시간으로 변하는 트랜젝션 갯수를 보는 것 만으로도 전체 시스템의 성능을 가능할 수 있습니다.

X-View

윈도우 타임내에 발생한 모든 트랜젝션을 처리 시간과 발생 시간을 기준으로 점을 찍어 표현합니다. 모든 트랜젝션이 2차원 그래프에 점으로 표현되기 때문에 어느 시간대에 느린 트랜젝션이 집중되는지를 한눈에 알아 볼수 있고, X-view에서 특정 구간을 마우스로 선택하면 해당 구간에 있는 모든 트랜젝션의 목록을 출력해 줍니다.

트랜젝션 통계 그래프

하룻동안 처리한 트랜젝션의 양과 평균 처리 시간을 선그래프로 표현합니다. 또한 과거 데이터를 함께 출력함으로써 과거와의 비교를 통해 현재의 상태를 더 직관적으로 파악 할 수 있습니다.

트랜젝션 상세 정보

하나의 트랜젝션에 대한 상세 정보를 출력합니다. 트랜젝션의 처리 시간, 발생 시간, 총 처리 과정등 다양한 정보를 세세하게 보여줍니다. 또한 Transaction Map을 통해 트랜젝션의 흐름을 한눈에 파악할 수 있도록 도와줍니다.

Call Stack

트랜젝션의 처리 과정을 코드 레벨에서 자세히 들여다 볼 수 있습니다. 어떤 함수처리에 시간이 얼마나 걸렸는지, 쿼리는 얼마나 걸렸는지 등 상당히 디테일한 개발자 수준의 정보를 볼 수 있습니다.

검색기능

다양한 검색 옵션을 통해 트랜젝션을 검색 할 수 있습니다. 특정 처리 과정 검색도 가능 하면 검색된 처리 과정이 어떤 트랜젝션에 속해 있는지 알 수 있고, 해당 트랜젝션에 대한 상세 정보를 열람 할 수 있습니다. 아무리 작은 단서라도 일단 검색에 걸리기만 하면 해당 처리과정 뿐만 아니라 처리 과정이 포함된 전체 트랜젝션 정보까지 모두 검색되어 효과적인 트러블 슈팅을 할 수 있도록 도와줍니다.

수동 추적

e2eMon은 자동으로 수집되는 정보로는 추적할 수 없는 데이터를 추적하기 위해 개발자가 직접 추적 정보를 추가할 수 있는 기능을 제공합니다. 개발자가 이 기능을 사용하면 거의 모든 정보가 추적 가능해지기 때문에 모니터링 범위가 비약적으로 확장될 수 있습니다.
