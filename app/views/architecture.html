시스템 아키텍쳐

e2eMon은 어떻게 동작하나요?

e2e Mon은 크게 3단계로 동작합니다.
모니터링 대상 시스템에서 추적에 필요한 정보를 수집하고 수집된 정보를 분류, 재가공하여 저장한뒤 저장된 정보를 가져와 Web Client로 보여줍니다.

각 단계에서 이루어지는 동작을 자세히 살펴보면 먼저 모니터링 대상 시스템에서 생성된 추적에 필요한 정보를 Log Collector가 파일에 적게 됩니다.
Log Stash는 해당 로그 파일을 감시하고 있다가 Log Collector가 정보를 파일에 적으면, 그걸 바로 읽어서 Message Queue(Kafka)로 실어 보내게 되고,
Message Queue Consumer는 Message Queue를 감시하고 있다가 Message가 들어오면 바로 읽어서 Search Engine (Elastic Search)에 저장하게 됩니다.
이렇게 일련의 단계를 거쳐 Search Engine에 저장된 정보는 다시 Log Composer가 주기적인 배치 작업을 통해 저장된 정보를 Transaction 별로 분류하고, Transaction 관련 통계 정보를 생성하는 과정을 수행합니다.
Log Composer에 의해 분류 및 재가공된 정보는 다시 용도에 따라 Search Engine(Elastic Search)과 RDB(PostgreSQL)에 나누어 저장되게 되고,
이렇게 저장된 정보는 API Server를 통해 Web Client로 전송되어 사용자에게 보여지게 됩니다.

e2eMon 실시간 플랫폼

  • Diagram 형태로 서버들 간의 Inbound/Outbound를 출력
  • Transaction 단위의 통계 정보를 출력
  • 검색조건에 맞는 Transaction을 리스트 형태로 출력
  • 해당 Transaction의 상세 내용을 출력
  • Kafka Flow 정보 설정 가능
  • 경고 알림을 위한 기준 정보 및 연락처 설정 가능
