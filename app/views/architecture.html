시스템 아키텍쳐

e2eMon은 어떻게 동작하나요?

e2eMon의 동작은 크게 4단계로 나눌 수 있습니다.

  • 기초로그 수집 - 모니터링 대상 시스템에서 추적에 필요한 기초 로그 정보를 수집
  • 기초로그 저장 - 수집된 기초 로그 정보를 1차 저장
  • Tx Log추출 & 저장 - 1차 저장된 정보를 분류, 재가공하여 Transaction Log의 형태로 2차 저장
  • Contents Display - 저장된 정보를 가져와 Web Client로 Display

1. 기초로그 수집

  • TxLogger(Bytecode Instrumentation + Library)를 이용해 Tracking Log를 File에 기록
  • Log Stash를 이용해 Log File에 적히는 Tracking Log를 실시간으로 수집 & Message Queue에 전송

2. 기초로그 저장

  • Consumer가 Message Queue에 있는 Log들을 가져와 Search Engine (Elastic)에 저장

3. Tx Log추출 & 저장

  • Elastic에 저장된 기초 Log를 가져와 Tx Log로 정제 후 다시 Search Engine (Elastic)에 저장
  • 에러 경보 프로세스 실행
  • Tx Log통계 연산 후 RDB (PostgreSQL)에 저장

4. Contents Display

  • Search Engin과 RDB에 저장된 정보를 API Server를 통해 가져와 Web Client에 Display
